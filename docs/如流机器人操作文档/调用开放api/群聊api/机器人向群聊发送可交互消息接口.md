### 接口请求体
请求体中采用json格式，主要包含以下内容，你也可以跳转最下方的[java的Vo类](https://ku.baidu-int.com/knowledge/HFVrC7hq1Q/2tsPs8CtSd/Bu7DDg4dpB/E_Ft-eJW7kbMH4?block_id=docyg-7fc51331-4126-11f0-9b49-07132fb78043&type=card)进行查看

|参数名|是否必填|数据类型|说明|
|-|-|-|-|
|appContext|否|任意json内容|上下文信息，用于回调时透传。最大长度限制XXX|
|header|是|json内容|请求头部信息，包括接受方id、提醒消息等。具体接受参数见下方。|
|body|否|json内容|对应卡片消息的正文区域。具体接受参数见下方。|
|panel|否|json内容，包含以下元素|卡片消息的交互区域|
|* buttonList|否|List<List<Object>>|Object具体格式参见下方<br/>最外层list允许最多8个元素，内层list允许3个元素。<br/>多余元素会被抛弃|
|* contactAndBindButtonList|否|List<Object>|Object具体格式参见下方。<br/>实际只允许一个按钮元素|

#### 关于按钮（buttonList）的补充说明
1、panel中支持一个buttonList，内部支持最多8个list，每个list最多支持3个button

2、eventType目前支持3类型，需要传入特定的字符串。根据传入的字符串，event中对应不同的json格式

|eventType|event|备注|
|-|-|-|
|"url"||用浏览器打开特定网站|
|"callBack"|{<br/>                "onlyChangePersonal": true // 用于控制千人一面和千人千面。为true时只有自己的界面数据发生变化 <br/>}|触发回调事件。onlyChangePersonal为true时，只修改个人界面的数据|
|"hijs"|{<br/>    "methodName":<br/>"BdHiJs.service.character.choose", <br/>    data: {} // data中的数据格式由methodName决定<br/>}|调用客户端的js，具体入参文档参考：[通用服务](https://ku.baidu-int.com/knowledge/HFVrC7hq1Q/2tsPs8CtSd/Bu7DDg4dpB/g7Ayt93zbYR1sM?t=mention&mt=doc&dt=doc)|

#### 关于成员选择器的补充说明
panel中目前支持最多一个contactAndBindButtonList，这是一个成员选择器以及和他唯一绑定的按钮。触发按钮事件后，回调链路能够拿到目前成员的唯一id，后续可结合@人等方式实现通知等功能。

                && Objects.nonNull(this.getHeader().getToid())
