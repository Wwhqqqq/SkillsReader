|-|-|-|-|-|
|其他参数|JsonObject|RequestBody|是|发送不同类型的消息使用不同的参数，详见[消息推送格式](https://ku.baidu-int.com/knowledge/HFVrC7hq1Q/2tsPs8CtSd/Bu7DDg4dpB/qYywDxy34JX-jc?t=mention&mt=doc&dt=doc)<br/>以AK/SK方式推送时，body中agentid字段不用设置，以AK对应的agentid为准。|

## **返回结果**
**示例：**

```json
{
    "errcode": 0,
    "errmsg": "ok",
    "msgkey": "123456789",
    "invaliduser": "UserID1",
    "invalidparty": "PartyID1",
    "invalidtag": "TagID1"
}
```
**参数说明**：

|参数|说明|
|-|-|
|errcode|返回码|
|errmsg|对返回码的文本描述内容|
|msgkey|消息mid|
|invaliduser|无效员工列表|
|invalidparty|无效部门列表|
|invalidtag|无效标签列表|

如果部分发送成功，，应答返回OK，可以在失败列表中，获取到未发送成功的名单

如果是全部发送失败，返回

```json
{
    "errcode": 40070,
    "errmsg": "接收者不在可见范围或未关注"
}
```




# 发送引用回复消息
## 各环境地址
preonline：http://xplatform-preonline.dev.weiyun.baidu.com/open-plat/api/devp/v1/app/message/send

灰度：http://apiin.im.baidu.com/gray/api/v1/app/message/send** **

线上：http://apiin.im.baidu.com/api/v1/app/message/send



## **请求头**
请求中需要包含以下请求头：

|**请求头名称**|**值/描述**|
|-|-|
|`Content-Type`|`application/json`|

## **查询参数**
|**参数名称**|**描述**|
|-|-|
|`access_token`|身份验证令牌（例如：`a04f7f95d256b09144fe38783d02fa532`）参考 [获取AccessToken](https://qy.baidu.com/doc/index.html#/inner_quickstart/flow?id=%E8%8E%B7%E5%8F%96accesstoken)|

## 请求体
|**字段**|**类型**|**是否必填**|**描述**|
|-|-|-|-|
|`touser`|`string`|`是`|接收消息的用户 ID（例如：`user_idxxx`）。|
|`toparty`|`string`|`否`|接收消息的部门 ID（如果不适用可留空）。|
|`totag`|`string`|`否`|接收消息的标签 ID（如果不适用可留空）。|
|`agentid`|`string`|`是`|发送方的 Agent ID（例如：`168`）。|
|`msgtype`|`string`|`是`|消息类型（例如：`text`）。|
|`text`|`object`|`是`|包含文本消息内容的对象。|
|`reply`|`array of object`|`否`|包含引用回复的数组，可选。|

```json

--header 'Content-Type: application/json' \
--header 'X-OP-AGENT-ID: 582' \
--header 'X-OP-CORP-ID: 1' \
--data '{
    "touser": "user_idxxx",
    "toparty": "",
```
