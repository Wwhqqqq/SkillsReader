整体功能：每天自动从多个平台（GitHub、SkillsMP、ClawHub、skills.sh 等）收集大量Skills，通过既定的筛选机制，挑选出最重要、最有价值的10个技能推荐给用户。
筛选标准包括：
* 是否来自大公司
* 是否最近突然爆火
* 是否质量高
* 是否有潜在趋势


6个步骤
第一步：数据收集
系统每天会定期（日期在控制台可选，以添加时间戳形式选择，可以一天两个三个时间）自动访问以下平台：

GitHub/SkillsMP/ClawHub/skills.sh
第二步：标准化处理
每个 Skill 会被转换成：
* 名称
* 来源平台
* 简介
* 链接
* 热度（stars / installs / views）
* 标签（如：腾讯 / AI / 工具）

第三步：去重
同一个 Skill 可能同时出现在多个平台：
例如：
* GitHub 有
* SkillsMP 有
* ClawHub 也有

如果不去重，会出现：
同一个技能被重复推荐3次（严重污染结果）
去重方法：
系统会生成一个“唯一指纹”：
判断依据包括：
* 项目名称
* GitHub地址
* Skill路径
* 平台ID

👉 去重结果：
* 重复内容合并成1条
* 保留最完整版本

七、第四步：候选池筛选（核心步骤）
由于技能太多，必须先筛选出“可能有价值的候选池”。
系统分4类池子：
🟡 1. 官方池（最高优先级）
包含：
* 腾讯官方 Skill
* 阿里官方能力
* 字节官方发布
* 美团官方工具
* 其他目前代码中设定的大公司的官方/ 个人开发者   （官方发布大于个人开发者，个人开发者如果命中大公司，要加分）

特点：
稳定 + 权威 + 必选关注

🟠 2. 热度池（流行技能）
包含：
* installs 很高
* stars 很高
* 使用人数快速增长

特点：
已经被市场验证

🔵 3. 趋势池（增长最快）
重点来源：
* skills.sh（核心）
* GitHub新项目增长
* SkillsMP短期爆发

特点：
可能“刚开始火”，但未来会爆

🟢 4. 发现池（新出现）
包含：
* 最近刚发布
* 还没火但质量不错
* 小众但结构完整

特点：
用于发现“潜力股”

八、第五步：评分系统（核心大脑）
每个 Skill 会被打一个“综合评分”。
评分来自四个方面：

⭐ 1. 官方程度（权重最高）
* 官方发布 → 加分最多
* 企业认证 → 次高
* 普通开发者 → 基础分


⭐ 2. 热度
衡量：
* 使用人数
* 点赞数量
* GitHub stars


⭐ 3. 增长速度（最重要的趋势指标）
看三种时间：
* 最近1天增长
* 最近3天增长
* 最近7天增长

👉 如果增长很快 = 说明正在爆发

⭐ 4. 内容质量
例如：
* 描述是否完整
* 是否有清晰说明
* 是否结构规范


九、统一“增长速度”的方法（关键难点）
因为不同平台指标不同：
平台
指标
GitHub
stars
SkillsMP
stars
ClawHub
installs
skills.sh
views

解决方案：
系统会做三步转换：

① 计算增长速度
例如：
* 今天100
* 昨天50

→ 增长100%

② 压缩极端值
避免某些爆火项目“碾压其他项目”

③ 转换为统一分数
让所有平台数据变成同一尺度

👉 最终效果：
不管来源是什么平台，都可以公平比较增长速度

十、最终排序逻辑（生成Top10）
系统不会简单排序，而是分结构推荐：

🥇 Top 1-3：官方重要技能
* 企业级能力
* 权威发布


🚀 Top 4-7：增长最快技能
* 最近爆发
* 潜在趋势


🌱 Top 8-10：新发现技能
* 小众但优质
* 可能未来爆发


十一、推送内容是什么样的？
每天推送的内容包括：
* 今日Top10 Skills
* skills简介
* 每个Skill为什么重要
* 来自哪个平台
* 是否官方
* 为什么推荐它
