下面给出一份完整的“Skills 智能每日精选推送系统设计文档”（面向用户+开发者双读版本），在你现有 Python 技术栈与多源 Adapter 架构基础上，进行系统级算法重构设计。我会严格围绕你当前代码体系（ClawHub / SkillsMP / GitHub / platform_filters / dedup / adapter）进行扩展，而不是另起炉灶。

一、系统总体目标与设计原则（核心框架）
本系统的核心目标不是“抓取 skills”，而是：

从高度冗余、跨平台、强噪声的 Skills 流中，每日自动筛选出“最值得关注的 Top10 Skills”。

这里的“值得关注”由三类信号构成：

① 供给信号（Supply Signal）
是否来自 ClawHub / SkillsMP / GitHub / 官方 OpenClaw

是否为大公司（腾讯/阿里/字节/美团等）

是否官方发布（metadata.official）

② 热度信号（Popularity Signal）
install_count（ClawHub / SkillsMP stars）

1日 / 3日 / 7日增长速度（trend velocity）

repo stars 增长（GitHub search补充）

③ 质量信号（Quality Signal）
SKILL.md 结构完整度

描述信息丰富度

是否被 filter_records_by_vendor_relevance 过滤通过

是否被 platform_filters 判定为“噪声”

系统设计原则：

✔ 原则1：多源不等权
ClawHub ≠ SkillsMP ≠ GitHub raw repo
必须统一归一化

✔ 原则2：不做全量监控
必须构建“候选池 + 动态扩展”，避免搜索爆炸

✔ 原则3：官方优先绝对加权
metadata.official 是硬优先级

✔ 原则4：增速优先于静态热度
“最近7天增长” > “历史stars”
